🕘 更新履歴

ArchView360の主なバージョンで、利用者の操作に影響する変更をまとめています。現在の対応バージョンは v2.22.0 です。

v2.20以前の変更について： VRモード（VRモード・Observer Mode）やプロジェクトZIP（プロジェクトJSON / プロジェクトZIP）など、v2.20までの機能追加は各トピックページ内にバージョン表記つきで解説があります。ここでは重複を避け、それらのページに未反映だったv2.21.0以降の変更を中心にまとめます。

Version 2.22.0

概要： 保存前にEditorでの編集内容が失われることを防ぐ「未保存の変更」管理を追加しました。あわせて、主要な操作を継続的に自動確認する仕組みを導入しました。

新機能：

  • ツールバーに「● 未保存」インジケーターを追加。編集中の内容がまだ保存されていない間、表示されます
  • ブラウザタブのタイトルにも、未保存の変更がある間は先頭に * が付きます
  • 未保存の変更がある状態でページを離れよう（閉じる・再読み込み）とすると、ブラウザ標準の確認が表示されます

変更：

  • Editorで未保存の変更がある状態からViewerへ切り替えようとすると、「保存せずViewerへ移動しますか」という確認が表示されます。「保存せずViewerへ移動」を選んでも編集内容は消えず、そのままメモリ上に残ります。Viewerで閲覧している間も未保存インジケーターは表示されたままで、Editorへ戻ればそのまま編集を続けられます
  • 設定JSON・プロジェクトZIPの書き出しが完了すると、未保存の状態は解除されます
  • すでに読み込んでいるプロジェクトに別のJSON・ZIPを読み込もうとすると（既存内容を置き換える操作のため）、「このまま続行すると保存していない変更が失われます」という確認が表示されます。「破棄して続行」を選んだ場合のみ、実際の読み込みが完了した後で未保存状態が解除されます
  • 空の状態（何も読み込んでいない状態）へ新しく画像・JSON・ZIPを開く操作は、確認なしでそのまま開けます

注意事項：

  • 未保存の状態はブラウザのタブを閉じたり再読み込みしたりすると失われます（意図的な設計です）。重要な変更は、確認ダイアログに従って保存してから次の操作を行ってください
  • ダイアログから直接保存する機能は今回のバージョンでは未対応です。保存する場合はキャンセルを選び、既存の「設定を書き出し」または「パッケージ書き出し」から保存してください
  • 品質保証・安定性向上： 主要な操作（起動確認・Viewer/Editor切替・保存後の状態確認など）を自動で確認する仕組みを追加し、コードの変更のたびにGitHub Actions上で自動テストが実行されるようにしました。リリースごとの動作確認の抜け漏れを減らすためのもので、画面の見た目や操作方法に変更はありません。

Version 2.21.0

概要： アプリ全体に「閲覧専用で使う」か「編集する」かを切り替えられるモードを導入しました。

新機能：

  • Viewerモード（閲覧専用） と Editorモード（編集可能） の2つのモードを追加。画面右上のボタンでいつでも切り替えられます
  • ArchView360を開いたとき・ページを再読み込みしたときは、常にViewerモードから始まります

UI変更：

  • Viewerモードでは、シーンの追加・削除・並び替え・画像の反転・平面図編集・マーカー配置・プロジェクト情報編集・設定の書き出し／読み込みなど、内容を変更する操作のボタンが非表示になります
  • Editorモードに切り替えると、これらの編集用ボタンがすべて表示され、これまでどおり編集できます
  • パノラマの視点操作・ズーム・シーン切替・比較モード・VRモードなど、内容を変更しない「見る」操作はViewer・Editorのどちらでも利用できます

操作方法の変更：

  • Viewerモードのままでも、まだ何も読み込んでいない状態であれば、単一画像・設定JSON・プロジェクトZIPを開いて閲覧することはできます（「開く」は閲覧の一部として扱われます）
  • すでに読み込んでいるプロジェクトへ画像を追加したり、内容を編集したりするにはEditorモードへの切り替えが必要です

互換性に関する注意：

  • モードの選択はそのブラウザタブ内だけで保持され、設定JSON・プロジェクトZIPには一切保存されません。別の端末やブラウザで開いた場合も、常にViewerモードから始まります
  • 既存のJSON・ZIPファイルの形式やVR・比較モードなどの閲覧機能に変更はありません
