🔒 セキュリティポリシー

ArchView360はサーバーを使わない、ブラウザ完結型のツールです。読み込んだデータがどこにも送信されない設計になっています。

データの取り扱い

データ外部送信保存先
パノラマ画像なしブラウザのメモリ内のみ
平面図（FloorMap）画像なしブラウザのメモリ内のみ
プロジェクトJSONなしユーザーが明示的に書き出した場合のみローカルファイルとして保存
比較セットなしブラウザのlocalStorage内のみ
Observer状態（v2.11：VR位置・視線方向・セッションID）なしブラウザのメモリ内のみ（プロジェクトJSONには保存されません）
ローカル完結型ツール： ArchView360にはサーバー側のバックエンドが存在しません。画像・平面図・JSONはすべてブラウザ内（クライアントサイド）で処理され、インターネット上のどこにもアップロードされません。

なぜ安全なのか

  • ファイル読み込みはブラウザの File API（ローカル処理）のみを使用しています。
  • ネットワーク通信を行うのは、アプリ本体（HTML/CSS/JS）とビューワーライブラリ（Three.js）の配信時のみです。
  • 読み込んだ画像・JSONを送信するAPIエンドポイントは存在しません。
  • プロジェクトの共有は、JSON＋画像ファイルをユーザー自身が任意の方法（メール・共有フォルダ等）で送付することで行います。

注意事項

ブラウザのタブを閉じる、またはページを再読み込みすると、読み込んだ画像やシーン構成は破棄されます（永続化されません）。重要なプロジェクトは「JSON書き出し」で随時保存してください。

プロジェクトZIPとデータ送信（v2.12）

「プロジェクトZIP」も、ブラウザ内（JSZipライブラリ、ローカル同梱）でそのままZIPを展開・生成するだけで、外部送信は一切行いません。ZIPの読み込み・書き出しのいずれもサーバーを経由しません。

VRモードとデータ送信

「VRモード」（WebXR単体VR表示）も、表示しているパノラマ画像をブラウザ内でそのまま描画するだけで、外部送信は一切行いません。VR表示中かどうかの状態もプロジェクトJSONには保存されません。v2.13で追加されたVR中のシーン切替・VR HUDも、コントローラー入力の処理とブラウザ内描画のみで完結し、外部送信は行いません。v2.14では入力取得方式をXRSession.inputSourcesの毎フレームポーリングに変更しましたが、これもブラウザ内のWebXR APIを直接参照するローカル処理であり、外部API通信・WebSocket・WebRTCは一切使用しません。HUDの表示状態・入力デバッグ情報（inputSources数・ボタン状態・入力カウンター等）はVRセッション内だけの一時状態で、プロジェクトJSON / ZIPには保存されません。

Observer Modeとデータ送信（v2.11）

「Observer Mode」は、現時点ではブラウザ内のローカル状態のみを扱う機能です。VR体験者の位置・視線方向・セッションIDはブラウザのメモリ内にのみ保持され、外部送信は行われません。プロジェクトJSONにも保存されません。将来Quest-PC間のネットワーク同期（WebSocket/WebRTC）を追加する場合は、その時点で本ページに明示します。
