📂 プロジェクトJSON / プロジェクトZIP

ArchView360はサーバーを使わないため、プロジェクトの状態（シーン構成・FloorMap・マーカー・グループ・比較セットなど）はJSONファイル、または画像も含めたZIPパッケージとして書き出し・読み込みします。

設定JSON

シーン構成・FloorMap・マーカー・グループ・比較セット・プロジェクト情報などの設定情報のみを含む軽量なJSONファイルです。画像データそのものはJSONには含まれません。

プロジェクトZIP（v2.12）

設定JSONに加えて、読み込み済みのパノラマ画像・平面図画像をすべて含めたZIPパッケージです。1ファイルだけで配布・受け渡しができ、社内展開や施主への共有に向いています。

ZIP構成例

ArchView360_Project_20260623_1530.zip
├─ project.json
├─ panoramas/
│   ├─ scene01.jpg
│   └─ scene02.jpg
├─ floorplans/
│   ├─ 1F.jpg
│   └─ 2F.jpg
└─ README.txt

使い分け

設定JSONプロジェクトZIP

向いている用途画像を別管理したい場合・差分更新社内展開・施主への一括共有・引っ越し
書き出しボタン「JSON書き出し」「📦 パッケージ書き出し」

ZIPの読み込み方

  1. 初期画面の「プロジェクトZIPを開く」ボタン、またはドロップゾーンへ .zip ファイルをドラッグ＆ドロップします。
  2. ビューア画面が表示されている状態でも、画面上に .zip をドラッグ＆ドロップすれば読み込めます。
  3. ZIP内の project.json が自動検出され、panoramas/・floorplans/ 内の画像がファイル名で自動照合・復元されます。
  4. シーン・FloorMap・マーカー・グループ・比較セット・プロジェクト情報がすべて復元されます。
  5. ZIP内に対応する画像が見つからない場合は、その分だけ復元されず、不足ファイル名がトースト通知で表示されます（可能な範囲で復元）。

ZIPの書き出し方

  1. 上部ツールバーの「📦 パッケージ書き出し」ボタンをクリックします。
  2. 現在のプロジェクト設定（project.json）、読み込み済みのパノラマ画像（panoramas/）、平面図画像（floorplans/）、README.txt がまとめてZIP化されダウンロードされます。
  3. ファイル名は ArchView360_Project_YYYYMMDD_HHMM.zip 形式です。

書き出す（設定JSON）

  1. サイドバーまたはツールバーの「プロジェクト情報」「JSON書き出し」操作からJSONファイルをダウンロードします。
  2. ファイルには各シーン・FloorMap・マーカー・グループ・比較セットの設定情報が含まれます（画像本体は含まれません）。

読み込む（プロジェクトを開く）

  1. トップ画面の「設定JSONを開く」から .json ファイルを選びます。
  2. JSONと画像ファイル一式を一緒にドラッグ＆ドロップすると、ファイル名で自動マッチングされ、シーン構成・FloorMap・マーカー・グループ・比較セットがすべて復元されます。
  3. 画像が見つからない場合は該当シーンが空のまま復元され、後から画像を再アサイン（または「🔄シーン更新」で差し替え）できます。
注意： 設定JSONはブラウザ内にのみ保存されるものではなく、ファイルとして任意の場所に保存されます。チームで共有する場合は、JSONファイルと画像ファイル一式をまとめて送付してください。
読み込み・書き出しはEditorモードで行います（v2.21〜）： まだ何も読み込んでいない状態であれば、Viewerモードのままでも設定JSON・プロジェクトZIPを開けます。すでに何か読み込んでいる状態へ別のJSON・ZIPを読み込む（置き換える）場合や、書き出しを行う場合はEditorモードへの切り替えが必要です。また、保存していない変更がある状態で別のプロジェクトを読み込もうとすると、「このまま続行すると、保存していない変更が失われます」という確認が表示されます（詳しくは「はじめに」の「未保存変更について」を参照）。

他のPC・ブラウザへの引っ越し

JSON＋画像一式があれば、別のPCやブラウザでも同じプロジェクトを完全に再現できます。比較セット（ローカル保存）はブラウザ内にのみ保存されるため、引っ越し先では別途保存し直してください。
