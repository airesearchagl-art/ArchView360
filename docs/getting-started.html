🚀 はじめに

ArchView360はブラウザだけで動く360°パノラマビューワーです。サーバーへのアップロードは行われず、選んだ画像はそのままブラウザ内（メモリ）で表示されます。

対応している画像

項目内容
形式JPEG / PNG / WebP
推奨投影方式正距円筒図法（Equirectangular）、推奨サイズ 4096×2048（2:1）
ファイルサイズ1ファイルあたり 100MB 以下
複数選択可（一度に複数のシーンを追加できます）

D5 Render など多くのレンダラーには「360°パノラマ（正距円筒図法）」での書き出しオプションがあります。書き出し時にこの形式を選んでください。

画像を読み込む

  1. トップ画面の点線エリアに画像ファイルをドラッグ＆ドロップする。
  2. またはエリア内の「ファイルを選択」ボタンからファイルダイアログで選ぶ。
  3. 読み込みが完了すると自動的にビューワー画面に切り替わります。

基本操作

  • 視点の操作： マウスでドラッグ（タッチ端末はスワイプ）して見回せます。
  • ズーム： マウスホイール、またはタッチ端末ではピンチ操作。
  • シーン切替： 左サイドバーのシーンをクリック、またはキーボードの ← →。
  • 視点リセット： ツールバーの「リセット」、またはキー R。
  • 左右反転： 鏡面合成などで反転している画像を直す場合に使用（キー M）。
  • VRモード： Meta QuestなどのWebXR対応ブラウザでは、上部ツールバーの「🥽 VRモード」ボタンで現在のシーンを没入表示できます。詳しくは「VRモード」を参照してください。
ヒント： 設定JSON（プロジェクトファイル）と画像をまとめてドロップすると、シーン構成・FloorMap・マーカー・グループなどをそのまま復元できます。詳しくは「プロジェクトJSON」を参照してください。

ZIPパッケージから開く（v2.12）

画像・平面図・設定をまとめた1つの .zip ファイルから、プロジェクトをまとめて復元できます。

  1. 起動画面の「プロジェクトZIPを開く」ボタンをクリック、または .zip ファイルをドロップゾーンにドラッグ＆ドロップします。
  2. ZIP内の project.json が自動検出され、panoramas/・floorplans/ フォルダ内の画像がファイル名で自動照合されます。
  3. 読み込みが完了すると、シーン・FloorMap・マーカー・グループ・比較セット・プロジェクト情報がすべて復元され、ビューワー画面に切り替わります。

詳しくは「プロジェクトJSON / プロジェクトZIP」を参照してください。

画面構成

ツールバー（画面上部）とシーン一覧（画面左のサイドバー）、中央の360°ビューワーで構成されます。ツールバーには「見るための操作」（自動回転・リセット・全画面・比較・VRなど）と「編集するための操作」（画像追加・シーン更新・左右反転・平面図・案件情報・書き出し／読み込みなど）が並びますが、後者は次に説明するEditorモードのときだけ表示されます。

Viewerモード / Editorモード（v2.21〜）

ArchView360には、画面右上のボタンで切り替えられる2つのモードがあります。ArchView360を開いたとき・ページを再読み込みしたときは、常にViewerモードから始まります。

Viewerモード（閲覧専用）

  • 視点の操作・ズーム・シーン切替・自動回転・全画面・分割比較／スライダー比較・VRモードなど、「見る」ための操作はすべて利用できます。
  • シーンの追加・削除・並び替え・画像の反転、平面図の追加・編集、マーカーの配置、案件情報の編集、設定の書き出し・読み込みなど、内容を変更する操作のボタンは表示されません。
  • まだ何も読み込んでいない状態であれば、Viewerモードのままでも単一画像・設定JSON・プロジェクトZIPを開いて閲覧できます（新しく開く操作は「見る」ことの一部として扱われます）。

Editorモード（編集可能）

  • 画面右上の「Editorへ切替」ボタンを押すと、Editorモードに入り、編集用のボタンがすべて表示されます。
  • シーンの追加・削除・並び替え・画像の反転・更新、平面図の追加・編集、マーカーの配置・移動、案件情報の編集、設定JSON・プロジェクトZIPの書き出し／読み込みなど、内容を変更する操作はEditorモードで行います。
  • Editorモードから「Viewerで確認」ボタンを押すと、Viewerモードへ戻ります。編集中の内容がまだ保存されていない場合の挙動は、次の「未保存変更について」を参照してください。

プロジェクトへ追加・インポートする

すでにシーンを読み込んでいる状態（プロジェクトが空でない状態）へ、さらに画像を追加したり、別の設定JSON・プロジェクトZIPを読み込んだりする場合は、Editorモードに切り替えてから行います。

  • 画像の追加： ツールバーの「画像追加」から、既存のシーンに新しい画像を追加できます。既存のシーンや設定は変更されません。
  • 設定JSON・プロジェクトZIPの読み込み（置き換え）： すでに何か読み込んでいる状態で別の設定JSON・プロジェクトZIPを読み込もうとすると、「このまま続行すると、保存していない変更が失われます」という確認が表示されます。保存していない変更がない場合は、そのまま置き換えが行われます。

編集する

Editorモードでは、これまでどおりシーンの並び替え・画像差し替え・平面図へのマーカー配置・比較セットの保存などが行えます。詳しい操作方法は「シーン管理」「FloorMap」「比較モード」を参照してください。

保存する

編集した内容は自動保存されません。ツールバーの「設定を書き出し」（JSON）または「パッケージ書き出し」（ZIP、画像も含む）から、その都度明示的に保存してください。書き出しが完了すると、次に説明する未保存インジケーターは消えます。詳しくは「プロジェクトJSON / プロジェクトZIP」を参照してください。

未保存変更について（v2.22〜）

Editorで何か変更を行うと、画面右上のApp Mode切替ボタンの隣に「● 未保存」という表示が現れ、ブラウザタブのタイトルの先頭にも * が付きます。これは、まだファイルとして保存していない変更があることを示します。

  • Viewerへ移動しても消えません： 未保存の変更がある状態で「Editorへ切替」の隣のボタンを押してViewerへ移動しようとすると、「保存せずViewerへ移動しますか」という確認が表示されます。「保存せずViewerへ移動」を選んでも、編集した内容が削除されるわけではありません。Viewerで閲覧を続けている間も未保存インジケーターは表示されたままで、Editorへ戻れば続きから編集できます。
  • 保存すると消えます： 「設定を書き出し」または「パッケージ書き出し」が完了すると、未保存インジケーターは消えます。
  • プロジェクトを置き換えると失われる可能性があります： 未保存の変更がある状態で、別のプロジェクトを開く・読み込む・「すべてクリア」を行おうとすると、「このまま続行すると、保存していない変更が失われます」という確認が表示されます。保存する場合は先にキャンセルし、書き出してから改めて操作してください。
  • 未保存の状態はそのブラウザタブの中だけで保持されます。タブを閉じたり、ページを再読み込みしたりすると失われるため、重要な変更は都度保存することをおすすめします。

エラー・復旧時の注意

表示中に何らかの理由で画面が正しく描画されなくなった場合（グラフィック処理の一時的な問題など）、ArchView360は自動的に画面をリセットして復旧を試みます。この自動復旧では、未保存の変更が誤って「保存済み」として扱われることはありません。復旧後に未保存インジケーターが表示されている場合は、引き続き保存されていない状態ですので、必要に応じて保存してください。

よくある操作上の注意

  • 編集用のボタンが見当たらない場合は、Viewerモードのままになっていないか画面右上を確認してください。
  • 未保存インジケーター（● 未保存）が表示されている間は、保存してから次の操作（プロジェクトを開く・切り替えるなど）に進むことをおすすめします。
  • 比較セットの保存・平面図へのマーカー配置など、Editorモード限定の操作は、ボタン自体がViewerモードでは表示されません。
