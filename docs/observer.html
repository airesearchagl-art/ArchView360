👁 Observer Mode（v2.11）

Meta QuestなどでVR体験している人が、現在どのシーンを見ていて、FloorMap上のどの位置で、どの方向を向いているかを、周囲の人がPC画面上で確認できる機能です。建築レビューなどでVR装着者だけが没入し、周囲が状況を把握しにくいという課題に対応します。

v2.11の対応範囲： 現時点では「ローカルObserver表示」のみに対応しています。Quest（Viewer）とPC（Observer）を実際にネットワーク越しでリアルタイム同期する機能は、将来のWebSocket / WebRTC対応として予定されています。

PC側 Observer と Quest側 Viewer の考え方

役割説明
Observer（観察者）PC側でFloorMapを見ながら、VR体験者の位置・視線方向を確認する側
Viewer（VR体験者）Quest BrowserなどでVRモードを使ってパノラマを体験している側

VRモードを開始すると、そのブラウザタブは自動的に Viewer（role: 'viewer'）として扱われます。Observerパネルから「Observer開始」を押すと、そのタブは Observer（role: 'observer'）として動作します。

想定利用シーン

  1. PC側： ArchView360を開き、プロジェクトJSONと画像一式を読み込み、Observer Modeを開始する
  2. Quest側： Quest BrowserでArchView360を開き、同じプロジェクトJSONと画像一式を読み込み、VRモードを開始する（Viewerとして接続）
  3. 結果： PC側のFloorMapに、Quest体験者の現在位置と視線方向がリアルタイムで表示される

Observerパネルの使い方

上部ツールバーの「👁 Observer」ボタンをクリックするとObserverパネルが開きます。表示内容は以下の通りです。

項目内容
状態停止中 / 待機中 / 接続中 / VR中（追跡中）
現在シーンVR/通常表示中の現在シーン名
FloorMap現在シーンが配置されているFloorMap名
マーカー現在シーンに対応するFloorMap上のマーカー（未配置の場合は「未配置」）
視線方向FloorMap上での視線方向（0〜360°）
FOV現在の視野角
セッションIDObserver開始時に自動生成されるID（将来のQuest-PC同期用）

ボタン：「Observer開始」「セッション終了」「Viewerとして接続」「セッションIDコピー」があります。

FloorMap上のObserverマーカー

Observer Modeが有効な間、通常のシーンマーカー（青）とは別に、VR体験者の位置を示す「Observerマーカー」が赤系の色で表示されます。

  • 色：赤／マゼンタ（通常のアクティブマーカーの青と区別できます）
  • 形：現在位置の丸＋視線方向を示す矢印
  • 視野角コーン：赤系半透明の扇形で、現在のFOVを示します
  • ラベル：「VR」表記
仕様： Observerマーカーは「通常マーカー → 通常FOVコーン」を描画した後、最後のレイヤーとして描画されるため、常に最前面に表示されます。

ローカルデモモード

Quest実機がない場合でも、Observer表示のUIを確認できるよう、現在のブラウザ内でローカルにObserver表示を確認できる「ローカルデモモード」を用意しています。

  • 上部ツールバーの「👁 Observer」→「Observer開始」を押すと、現在表示中のシーンに対応するマーカーが赤いObserverマーカーとして表示されます
  • 通常表示でマウス／タッチ操作によりドラッグすると、視線方向（yaw）が更新され、赤い矢印・FOVコーンが追従します
  • VR中はHMDの向き（カメラ姿勢）から視線方向が更新されます
  • シーンを切り替えると、Observerパネルとマーカー位置が自動的に更新されます

VR中の視線方向トラッキング

VRモード中は、Three.jsのWebXRカメラ（HMDの姿勢を反映する camera）の回転（camera.quaternion / camera.matrixWorld）から正面方向ベクトルを取得し、FloorMap上の視線方向（度数）に変換してObserver状態へ反映します。負荷軽減のため、約10回/秒の間隔で更新されます。

注意： この処理はObserver表示専用の一時的な値であり、マーカー自体のrotationプロパティを上書きすることはありません。Observerの状態（位置・視線方向・セッションIDなど）はプロジェクトJSONには一切保存されません。

シーン切替・未配置シーンの扱い

Observer Mode中にシーンを切り替えると、対応するFloorMap上のマーカーが自動的に探索され、Observerパネルとマーカー表示が更新されます。シーンがどのFloorMapにも配置されていない場合は、Observerパネルに「このシーンはFloorMap未配置です」という警告が表示され、FloorMap上のObserverマーカーは表示されません。

v2.13： VR中にQuest Touch Plusコントローラーでシーンを切り替えた場合（VR Scene Navigation、入力方式はv2.15でWebXR-Sandbox実証済みのsession.inputSourcesポーリング方式に刷新）も同様に、Observer状態（シーン・マーカー・FloorMap・視線方向）が自動更新され、FloorMap上の赤いObserverマーカーが新しいシーンの位置へ追従します。

Quest-PC同期の現時点の制限

  • v2.11では、Observer表示は同一ブラウザタブ内のローカル状態のみを反映します（同一端末・同一ブラウザでのデモ用途）
  • Quest側とPC側を別端末でリアルタイムに同期する機能（ネットワーク越しの位置・視線方向共有）は、今回のバージョンでは実装していません
  • 「Viewerとして接続」ボタンはUIとして用意していますが、実際のネットワーク同期は行われず、「ローカル同期は今後対応予定です」と表示されます
  • 将来的にはWebSocketまたはWebRTCを使った同一ネットワーク内同期、さらにクラウド経由の共有（Share360連携等）への拡張を検討しています

セキュリティ方針

Observer Modeにおいても、ArchView360のローカル完結方針は変わりません。

  • v2.11では外部送信は一切行われません
  • Observer状態（位置・視線方向・セッションIDなど）はブラウザ内のメモリ上にのみ保持されます
  • プロジェクトJSONには保存されません
  • 将来ネットワーク同期機能を追加する場合は、その時点で本マニュアルおよびセキュリティポリシーに明示します
