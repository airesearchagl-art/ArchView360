# ArchView360

> **v2.4** — D5 Renderなどで書き出した360°パノラマ画像をブラウザ上で確認できる、軽量な静的Webビューワーです。複数シーンの管理・2画面比較・スライダー比較・ドラッグ並び替え・比較セット保存・FloorMap Navigator（視野角コーン・マーカー名称・マーカー一覧・情報パネル）・JSONプロジェクト復元に対応し、建築プレゼンや設計レビューに使えます。

## サムネイル付きシーン一覧

アップロードした各パノラマ画像のサムネイルを自動生成してサイドバーに表示します。

- **自動生成**: ブラウザ内の Canvas API で 240×135px のプレビュー画像を生成（サーバー送信なし）
- **ホバー拡大**: サムネイルにマウスを乗せると 240×135px の拡大プレビューをサイドバー横に表示
- **アクティブ強調**: 現在表示中のシーンは青い枠で強調
- **クリック切替**: サムネイルをクリックするとそのシーンに切替

## ドラッグ＆ドロップ並び替え

サイドバーのシーンをドラッグして並び替えることができます。

```
使い方:
1. 並び替えたいシーンをドラッグ
2. 挿入したい位置に移動（挿入線が表示される）
3. ドロップで並び替え完了

並び替え後:
- シーン番号が自動更新される
- 現在表示中のシーンはそのまま維持される
- 比較A/Bに選択されているシーンも追跡される
- ← → キーの移動順も並び替え後の順序に従う
```

### 建築用途の並び替え例

```
並び替え前（アップロード順）:
  1. 3F_執務室.jpg
  2. 1F_エントランス.jpg
  3. 外観_東.jpg
  4. 2F_食堂.jpg

プレゼン順に並び替え後:
  1. 外観_東.jpg      ← 外から確認
  2. 1F_エントランス  ← 入口の雰囲気
  3. 2F_食堂          ← 共用スペース
  4. 3F_執務室        ← 主要執務スペース
```

## 比較モードの使い方

### 分割比較

```
1. 2つ以上のシーンを読み込む
2. 上部ツールバー「⊞ 分割比較」をクリック（またはCキー）
3. A・Bのドロップダウンで比較対象を選択
4. ドラッグで視点移動（視点同期ONのとき両面が連動）
5. 「← 通常表示」ボタンまたはEscで通常表示へ戻る
```

### スライダー比較

```
1. 2つ以上のシーンを読み込む
2. 上部ツールバー「◫ スライダー比較」をクリック（またはSキー）
3. 中央のスライダーをドラッグして境界を移動
4. 左側がA、右側がB（中央ハンドルで境界を調整）
```

### 視点同期 ON/OFF (Lキー)

```
視点同期 ON（デフォルト）:
  - AをドラッグするとBも同じ視点で動く
  - 同じアングルで仕上げ違いを比較できる

視点同期 OFF:
  - AとBを別々のアングルで操作できる
  - 「部屋の奥から見た案A」と「廊下から見た案B」を同時確認
  - 異なる視点の確認に使える
```

### 建築用途の比較例

| ユースケース | シーン A | シーン B |
|-------------|---------|---------|
| 壁あり / なし比較 | 壁あり案 | 壁なし案 |
| 仕上げ違い比較 | タイル仕上げ | 塗装仕上げ |
| 昼景 / 夕景 | 昼間レンダリング | 夕景レンダリング |
| 提案 A / B | 提案Aリビング | 提案Bリビング |
| 同視点での確認 | 視点同期ON | 同一アングル比較 |
| 別方向の確認 | 視点同期OFF | Aはエントランス、Bは廊下 |

## キーボードショートカット

| キー | 操作 |
|------|------|
| `←` | 前のシーンへ（通常表示のみ） |
| `→` | 次のシーンへ（通常表示のみ） |
| `R` | 視点リセット |
| `A` | 自動回転 ON / OFF |
| `F` | 全画面切り替え |
| `M` | 左右反転（通常表示のみ） |
| `C` | 分割比較 ON / OFF |
| `S` | スライダー比較 ON / OFF |
| `V` | 分割比較中：左右 ↔ 上下切替 |
| `L` | 比較中：視点同期 ON / OFF |
| `Esc` | 通常表示に戻る |

## D5 Render 推奨書き出し設定

```
レンダリング種別  : パノラマ（360°レンダリング）
投影方式         : 正距円筒図法（Equirectangular）  ← 必須
縦横比           : 2 : 1  ← 必須
推奨解像度        : 4096 × 2048 px
```

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript（外部依存なし）
- [Three.js r128](https://threejs.org/) — `vendor/three.min.js` としてローカル同梱
- 画像はブラウザ内で処理（`URL.createObjectURL()`）。**サーバーへ送信されません**

---

## 注意点

- 非常に高解像度（8K以上）の画像はブラウザのメモリを大量に消費します
- iOS Safari では WebP の対応が限定的な場合があります。JPEG を推奨します
- `file://` プロトコルより `npx serve .` などのローカルサーバー経由を推奨します

## FloorMap Navigator の使い方

```
1. ツールバー「🗺 平面図」→ 平面図画像をアップロード
2. 画面右下に FloorMap Navigator が表示される
3. 「📍」ボタンでマーカー配置モードON
4. 平面図上をクリック → 現在表示中のシーンのマーカーを配置（現在の視点方向を自動取得）
5. マーカークリック → そのシーンへ移動（配置モード・通常モード両方で動作）
6. マーカードラッグ → 位置を移動
7. マーカー選択後、情報パネルに名称・シーン・向きが表示される
8. ✏ボタンまたはダブルクリック → マーカー名称をインライン編集
9. ↺/↻ で15°刻みで向きを調整
10. 視野角コーン（扇形）がアクティブマーカーの向きとFOVをリアルタイムで表示
11. マーカーにホバーするとツールチップで名称とシーン名を表示
12. パネル下部のマーカー一覧でシーン移動・現在位置ハイライト
13. パネル上端をドラッグして高さを自由調整、または S/M/L/XL ボタンで幅プリセット切替
14. − で折りたたみ
15. 複数平面図を追加すると切替ドロップダウンが表示される
16. 360°ビューをドラッグ中はマーカーの矢印がリアルタイムで更新される
```

## マーカーの視野角コーン仕様

- アクティブ（現在シーン）のマーカーのみ扇形コーンを表示
- コーン角度 = 現在のFOV（ホイールズームに連動）
- コーン長さ = 55px（固定）
- 色 = 半透明ブルー（fill: rgba(79,124,255,0.15)）
- リアルタイム更新: マウスドラッグ・ホイールズーム・リセット・シーン切替すべてに追従
- 左右反転シーンも対応（thetaToFloorRotation の符号で吸収）
- コーンはJSONに保存されない（表示専用）

## JSON Export / Import の使い方

### 推奨フォルダ構成

```
Project_360/
├─ archview360-project.json
├─ floorplans/
│   ├─ 1F.jpg
│   └─ 2F.jpg
└─ panoramas/
    ├─ A-1.jpg
    ├─ A-2.jpg
    └─ Entrance.jpg
```

### Export

```
ツールバー「⬇ 書き出し」→ archview360-project.json をダウンロード
※ JSONには画像データが含まれません（ファイル名のみ）
※ 画像は別途フォルダで管理してください
```

### Import（別PCで復元する場合）

```
1. ツールバー「⬆ 読み込み」→ archview360-project.json を選択
2. 必要なファイル一覧が表示される
3. 「画像ファイルを選択して復元」→ 平面図＋パノラマ画像を一括選択
4. fileName が完全一致したファイルのみ復元される
```

> ⚠️ 別PCで復元するには画像ファイル一式が必要です。JSONだけでは画像を復元できません。

### JSON スキーマ（v2.2）

```json
{
  "appVersion": "2.0",
  "exportedAt":  "...",
  "projectName": "...",
  "scenes":      [{ "id", "name", "fileName", "flipped" }],
  "floorplans":  [{ "id", "name", "fileName" }],
  "markers":     [{ "id", "floorplanId", "sceneId", "x", "y", "rotation" }],
  "compareSets": [...]
}
```

`x` / `y` は画像内の比率（0〜1）で保存。`imageUrl` フィールドは将来の ArchShare360 対応のため予約済み。

## 比較セット保存の使い方

```
1. 分割比較またはスライダー比較モードに入る
2. A・Bシーンを選択、レイアウト・スライダー位置を調整
3. ツールバー「💾 セット保存」をクリック
4. アプリ内モーダルでセット名を入力して「保存」
5. 左サイドバーの「比較セット」欄にカード表示される
6. カードをクリックまたは「開く」ボタンで比較設定を復元
7. ✏ で名前変更（モーダル）、× で削除
```

### 保存場所と制約

| 項目 | 内容 |
|------|------|
| 保存場所 | このブラウザの localStorage |
| 画像データ | **保存されません**（シーン名・設定のみ） |
| 共有 | **別PCや別ブラウザには共有されません** |
| 削除 | セットが参照するシーンを削除した場合は復元時にエラー通知 |

## 初期画面から設定JSONを開く

### 方法A：ボタンから開く

```
1. 起動画面の「設定JSONを開く」ボタンをクリック
2. archview360-project.json を選択
3. 必要ファイル一覧（パノラマ・平面図）が表示される
4. 「画像ファイルを選択して復元」で画像ファイルを一括選択
5. fileName が完全一致したものだけ復元される
6. トーストにシーン/平面図/マーカー/比較セット数が表示される
```

### 方法B：JSONをドロップゾーンにドラッグ＆ドロップ

```
ケース1 — JSONのみドロップ:
  1. .json ファイルをドロップゾーンにドラッグ＆ドロップ
  2. 復元モーダルが表示される
  3. 「画像ファイルを選択して復元」で画像を選択

ケース2 — JSON + 画像を同時ドロップ:
  1. .json と画像ファイルをまとめてドロップゾーンにドラッグ＆ドロップ
  2. JSON内の fileName と画像ファイル名を自動照合
  3. 一致する画像を使ってそのまま復元される（モーダル省略可能）
  4. 不足ファイルがある場合はその分だけ復元されない

ケース3 — 画像のみドロップ:
  → 従来通り新規プロジェクトとして開始
```

> ⚠️ 別PCで復元するには画像ファイル一式が必要です。JSONだけでは画像を復元できません。**ファイル名は完全一致**が必要です（大文字小文字も含む）。

## FloorMap ピン方位仕様

- パノラマビューを右方向に回すと、FloorMapのピン矢印も右方向に回転します
- パノラマビューを左方向に回すと、ピン矢印も左方向に回転します
- 左右反転（flipH=true）シーンでも同じ方向感で動作します
- ピン配置時（クリック）も現在の視線方向を自動取得して矢印方向を設定します
- ↺/↻ ボタンで15°単位の手動調整が可能です

## キーボードショートカット
