# ArchView360

> **v1.8** — D5 Renderなどで書き出した360°パノラマ画像をブラウザ上で確認できる、軽量な静的Webビューワーです。複数シーンの管理・2画面比較・スライダー比較・ドラッグ並び替え・比較セット保存に対応し、建築プレゼンや設計レビューに使えます。

## クイックスタート

```bash
git clone https://github.com/airesearchagl-art/ArchView360.git
cd ArchView360
npx serve .   # → http://localhost:3000
```

> Three.js は `vendor/three.min.js` としてローカル同梱されているため、**オフラインでも動作します**。

## サムネイル付きシーン一覧

アップロードした各パノラマ画像のサムネイルを自動生成してサイドバーに表示します。

- **自動生成**: ブラウザ内の Canvas API で 240×135px のプレビュー画像を生成（サーバー送信なし）
- **ホバー拡大**: サムネイルにマウスを乗せると 240×135px の拡大プレビューをサイドバー横に表示
- **アクティブ強調**: 現在表示中のシーンは青い枠で強調
- **クリック切替**: サムネイルをクリックするとそのシーンに切替

## ドラッグ＆ドロップ並び替え

サイドバーのシーンをドラッグして並び替えることができます。

```
使い方:
1. 並び替えたいシーンをドラッグ
2. 挿入したい位置に移動（挿入線が表示される）
3. ドロップで並び替え完了

並び替え後:
- シーン番号が自動更新される
- 現在表示中のシーンはそのまま維持される
- 比較A/Bに選択されているシーンも追跡される
- ← → キーの移動順も並び替え後の順序に従う
```

### 建築用途の並び替え例

```
並び替え前（アップロード順）:
  1. 3F_執務室.jpg
  2. 1F_エントランス.jpg
  3. 外観_東.jpg
  4. 2F_食堂.jpg

プレゼン順に並び替え後:
  1. 外観_東.jpg      ← 外から確認
  2. 1F_エントランス  ← 入口の雰囲気
  3. 2F_食堂          ← 共用スペース
  4. 3F_執務室        ← 主要執務スペース
```

## 比較モードの使い方

### 分割比較

```
1. 2つ以上のシーンを読み込む
2. 上部ツールバー「⊞ 分割比較」をクリック（またはCキー）
3. A・Bのドロップダウンで比較対象を選択
4. ドラッグで視点移動（視点同期ONのとき両面が連動）
5. 「← 通常表示」ボタンまたはEscで通常表示へ戻る
```

### スライダー比較

```
1. 2つ以上のシーンを読み込む
2. 上部ツールバー「◫ スライダー比較」をクリック（またはSキー）
3. 中央のスライダーをドラッグして境界を移動
4. 左側がA、右側がB（中央ハンドルで境界を調整）
```

### 視点同期 ON/OFF (Lキー)

```
視点同期 ON（デフォルト）:
  - AをドラッグするとBも同じ視点で動く
  - 同じアングルで仕上げ違いを比較できる

視点同期 OFF:
  - AとBを別々のアングルで操作できる
  - 「部屋の奥から見た案A」と「廊下から見た案B」を同時確認
  - 異なる視点の確認に使える
```

### 建築用途の比較例

| ユースケース | シーン A | シーン B |
|-------------|---------|---------|
| 壁あり / なし比較 | 壁あり案 | 壁なし案 |
| 仕上げ違い比較 | タイル仕上げ | 塗装仕上げ |
| 昼景 / 夕景 | 昼間レンダリング | 夕景レンダリング |
| 提案 A / B | 提案Aリビング | 提案Bリビング |
| 同視点での確認 | 視点同期ON | 同一アングル比較 |
| 別方向の確認 | 視点同期OFF | Aはエントランス、Bは廊下 |

## キーボードショートカット

| キー | 操作 |
|------|------|
| `←` | 前のシーンへ（通常表示のみ） |
| `→` | 次のシーンへ（通常表示のみ） |
| `R` | 視点リセット |
| `A` | 自動回転 ON / OFF |
| `F` | 全画面切り替え |
| `M` | 左右反転（通常表示のみ） |
| `C` | 分割比較 ON / OFF |
| `S` | スライダー比較 ON / OFF |
| `V` | 分割比較中：左右 ↔ 上下切替 |
| `L` | 比較中：視点同期 ON / OFF |
| `Esc` | 通常表示に戻る |

## D5 Render 推奨書き出し設定

```
レンダリング種別  : パノラマ（360°レンダリング）
投影方式         : 正距円筒図法（Equirectangular）  ← 必須
縦横比           : 2 : 1  ← 必須
推奨解像度        : 4096 × 2048 px
```

## デプロイ

### Vercel（推奨）

1. [Vercel](https://vercel.com/) にログイン
2. **「Add New Project」** → GitHubリポジトリを選択
3. ビルドコマンド・出力ディレクトリは**空欄のまま**でOK
4. **「Deploy」** をクリック → 数分で公開完了

### GitHub Pages

Settings → Pages → Source を `main` ブランチの `/(root)` に設定

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript（外部依存なし）
- [Three.js r128](https://threejs.org/) — `vendor/three.min.js` としてローカル同梱
- 画像はブラウザ内で処理（`URL.createObjectURL()`）。**サーバーへ送信されません**

---

## 注意点

- 非常に高解像度（8K以上）の画像はブラウザのメモリを大量に消費します
- iOS Safari では WebP の対応が限定的な場合があります。JPEG を推奨します
- `file://` プロトコルより `npx serve .` などのローカルサーバー経由を推奨します

## 比較セット保存の使い方

```
1. 分割比較またはスライダー比較モードに入る
2. A・Bシーンを選択、レイアウト・スライダー位置を調整
3. ツールバー「💾 セット保存」をクリック → セット名を入力
4. サイドバーの「比較セット」欄に保存される
5. セット名をクリックすると比較設定を復元
6. ✏ で名前変更、× で削除
```

> 比較セットはブラウザの localStorage に保存されます。  
> セットが参照するシーンを削除した場合は、復元時にエラーが通知されます。
