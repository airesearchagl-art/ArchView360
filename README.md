# ArchView360

> **v1.1** — D5 Renderなどで書き出した360°パノラマ画像をブラウザ上で確認できる、軽量な静的Webビューワーです。複数シーンの管理に対応し、建築プレゼンやフロア確認に使えます。

[![Three.js](https://img.shields.io/badge/Three.js-r128-green)](https://threejs.org/)
[![License](https://img.shields.io/badge/license-MIT-lightgrey)](LICENSE)

## クイックスタート

### ローカルで開く

```bash
git clone https://github.com/airesearchagl-art/ArchView360.git
cd ArchView360
# index.html をブラウザで直接開くだけで動作します
# または簡易サーバーを起動（推奨）
npx serve .
# → http://localhost:3000
```

> Three.js を CDN から読み込むため、**初回はインターネット接続が必要**です。

### 複数シーンの読み込み

```
1. 複数の360°パノラマ画像をまとめてドラッグ＆ドロップ
   → 左サイドバー（モバイルは下部）にシーン一覧が表示されます

2. シーンをクリックして切替
   ← → キーでも前後に移動できます

3. シーン名をダブルクリックして編集（Enter で確定、Esc でキャンセル）

4. ＋ ボタンで後からシーンを追加
   × ボタンで個別シーンを削除
```

### 操作方法

| 操作 | 内容 |
|------|------|
| ドラッグ | 視点を移動 |
| スクロール | ズームイン／アウト |
| スワイプ | 視点を移動（タッチ） |
| ピンチ | ズーム（タッチ） |
| ↺ リセット | 初期視点・FOVに戻る |
| ▶ 自動回転 | 自動回転 ON / OFF |
| ⛶ 全画面 | 全画面表示に切り替え |
| 📂 戻る | すべてクリアしてアップロード画面へ |

### キーボードショートカット（ビューワー表示中）

| キー | 操作 |
|------|------|
| `←` | 前のシーンへ |
| `→` | 次のシーンへ |
| `R` | 視点リセット |
| `A` | 自動回転 ON / OFF |
| `F` | 全画面切り替え |

> シーン名を編集中はショートカットが無効になります。

## 建築用途の使用例

### フロア別確認

```

→ 4ファイルをまとめてドロップ → ← → キーでフロアを切替

```

## D5 Render 推奨書き出し設定

```
レンダリング種別  : パノラマ（360°レンダリング）
投影方式         : 正距円筒図法（Equirectangular）  ← 必須
縦横比           : 2 : 1  ← 必須
```

### 推奨解像度

| 用途 | 解像度 | ファイルサイズ目安 |
|------|--------|-----------------|
| 動作確認・プレビュー | 2048 × 1024 px | 〜 2 MB |
| **標準・ポートフォリオ（推奨）** | **4096 × 2048 px** | **〜 8 MB** |
| 高精細・最終確認 | 8192 × 4096 px | 〜 30 MB |

### ファイル形式の選び方

| 形式 | 推奨シーン |
|------|-----------|
| **JPEG** | 標準。ファイルサイズが小さく全ブラウザ対応 ✅ |
| PNG | 透過・高品質が必要な場合。ファイルサイズ大 |
| WebP | iOS Safari での対応が限定的なため非推奨 ⚠️ |

> **注意:** 縦横比が 2:1 以外（例：正方形・ワイドすぎる画像）の場合、球体への貼り付けが歪みます。

## デプロイ方法

### Vercel（推奨）

**方法 A — ダッシュボード（最短）:**

1. [Vercel](https://vercel.com/) にログイン
2. **「Add New Project」** → GitHub リポジトリを選択
3. ビルドコマンド・出力ディレクトリは**空欄のまま**でOK
4. **「Deploy」** をクリック → 数分で公開完了

**方法 B — CLI:**

```bash
npm i -g vercel
cd ArchView360
vercel --prod
```

**Vercel デプロイ後の確認手順:**

```
1. ルートURL（https://xxx.vercel.app/）にアクセス
   → アップロード画面が表示されること

2. テスト用パノラマ画像（JPEG または PNG）をドラッグ＆ドロップ
   → ローディング表示の後、360°ビューが表示されること
   → 「ドラッグで自由に見回せます」トーストが表示されること

3. マウスドラッグで視点移動できること

4. ホイールで FOV が変化すること（ステータスバーの数値が変わる）

5. 「自動回転」ボタンをクリック → ゆっくり回転開始

6. 「全画面」ボタンをクリック → 全画面に切り替わること
   （iOS Safari では全画面ボタン非表示が正常）

7. 「画像変更」ボタン → アップロード画面に戻り、再度画像が選べること

8. ブラウザ DevTools Console に赤いエラーがないこと
```

### GitHub Pages

1. リポジトリの **Settings → Pages** を開く
2. Source を `main` ブランチの `/ (root)` に設定
3. 数分後に `https://<username>.github.io/ArchView360/` で公開

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript（外部依存パッケージなし）
- [Three.js r128](https://threejs.org/) — CDN 経由で読み込み
- `vercel.json` — キャッシュ・セキュリティヘッダー設定済み

### 実装の要点

- `SphereGeometry` に `side: THREE.BackSide` を指定して球体内面描画（正距円筒図法の正しい投影方法）
- `URL.createObjectURL()` でローカルファイルを読み込み（サーバーへの送信なし）
- 切り替え時に `geometry.dispose()` / `material.map.dispose()` / `URL.revokeObjectURL()` で完全メモリ解放
- Fullscreen API の webkit prefix を Safari 向けに対応
- WebGL コンテキストロスト時のエラー復帰フロー実装

## v1.1 — 複数シーン管理

### 複数画像アップロード

ドラッグ＆ドロップまたはファイル選択で複数ファイルを一括読み込みできます。追加後は左サイドバー（モバイルでは下部バー）にシーン一覧が表示されます。

### シーンパネル

- **番号と名前**：各シーンに連番と名前が表示されます
- **アクティブ表示**：現在表示中のシーンが青くハイライト
- **追加（＋）**：シーンを後から追加
- **クリア（🗑）**：全シーンを一括削除してアップロード画面へ戻る

### シーン名編集（ダブルクリック）

シーン名をダブルクリックするとインライン編集できます。`Enter` またはフォーカス移動で保存、`Escape` でキャンセルします。

### シーン削除

各シーン行の `×` ボタンで個別削除できます。テクスチャ・BlobURL を完全に解放するためメモリリークが起きません。

### キーボードショートカット

| キー | 動作 |
|------|------|
| `←` | 前のシーンへ |
| `→` | 次のシーンへ |
| `R` | 視点リセット |
| `A` | 自動回転 ON / OFF |
| `F` | 全画面切り替え |

> シーン名を編集中（contenteditable）またはテキスト入力中はショートカットが無効になります。

### 活用シーン例

| ユースケース | シーン構成例 |
|-------------|------------|
| フロア確認 | 1F・2F・3F を各シーンに登録 |
| インテリア比較 | パターン A / パターン B を並べて比較 |
| 建物外観＋室内 | 外観・エントランス・ダイニング・オフィスを1セットで確認 |

## 注意点

- **オフライン環境** では Three.js CDN が読み込めないため動作しません。オフライン利用が必要な場合は `three.min.js` をローカルに配置してください。
- 非常に高解像度（8K 以上）の画像はブラウザのメモリを大量に消費することがあります。
- iOS Safari では WebP の対応が限定的な場合があります。JPEG 形式を推奨します。
- 画像はサーバーへ送信されません。すべての処理はブラウザ内で完結します。
- `file://` プロトコルで直接開いた場合も動作しますが、一部ブラウザでは制限があります。ローカルサーバー（`npx serve .`）経由を推奨します。
